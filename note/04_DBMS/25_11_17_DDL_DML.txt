3장 DDL & DML
	DCL : 사용자 권한 제어
    		사용자계정 생성 CREATE USER, 권한부여 GRANT
    		권한박탈 REVOKE, 사용자계정삭제 DROP USER
    		트렌젝션 명령어(ROLLBACK, COMMIT)
	DDL : 데이터 구조를 정의
    		테이블생성 CREATE TABLE, 테이블구조변경 ALTER TABLE, 테이블삭제 DROP TABLE
	DML : CRUD
    		INSERT 입력, SELECT 검색, UPDATE 수정, DELETE 삭제
    		DML 명령어는 취소가 가능
 3.1 DDL
	테이블 생성 (CREATE TABLE 테이블명...) : 테이블의 구조 정의
	테이블 삭제 (DROP TABLE 테이블명)
 3.2 DML
	데이터 삽입
		INSERT INTO 테이블명 (필드명1, 필드명2, ...) VALUES (값1, 값2, ....);
		INSERT INTO 테이블명 VALUES (값1, 값2, ... )
			필드명을 사용하지 않을시 값들은 테이블의 필드 순서대로
	데이터 변경
		UPDATE 테이블명 SET 필드명1=값1[, 필드명2=값2, ..., 필드명n=값n] [WHERE 조건];
	데이터 삭제
		DELETE FROM 테이블명 [WHERE 조건];
 3.3 제약조건
	PRIMARY KEY : 테이블의 각 행을 유일한 값으로 식별하기 위한 필드
	FOREIGN KEY : 테이블의 열이 다른 테이블의 열을 참조
	NOT NULL : NULL을 포함하지 않는다
	UNIQUE : 모든 행의 값이 유일해야, NULL값은 허용(NULL은 여러개 가능)
	CHECK(조건) : 해당 조건이 만족 (NULL값 허용)
	DEFAULT 기본값 : 기본값 설정(해당 열의 데이터를 입력하지 않고 INSERT하면 NULL이 들어갈 것을 DEFAULT값이 들어가도록)
		DEFAULT의 경우 제약 조건은 아니지만 제약 조건과 동일하게 사용가능
